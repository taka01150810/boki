# 日商簿記 仕訳 一問一答 — 設計ドキュメント

## 概要

日商簿記**2級・3級**の仕訳問題を1問ずつ解いて答え合わせできる学習アプリ。
取引文を読み、借方・貸方の勘定科目（プルダウン）と金額（入力）を埋めて「解答する」を押すと、自動採点して正誤と正解の仕訳を表示する。

## 参考にしたサイト

- **いぬぼき 2級 仕訳問題** — <https://inuboki.com/2q-siwake/>
  出題形式（1問ずつ取引文を提示 → 借方・貸方を科目プルダウン＋金額で入力 → 答え合わせ）は、このサイトの仕訳一問一答を参考にしている。
  収録している2級190問の出典でもある（各 md の冒頭に URL を記載）。
- 現在収録している3級15問の出典は [3級/boki3-workbook-01.md](./3級/boki3-workbook-01.md) に記載。

## 主要な設計判断（インタビューで確定）

| 項目 | 決定 | 理由 |
|---|---|---|
| 中核形式 | 仕訳問題（取引文 → 借方/貸方） | 手元の3級仕訳データをそのまま活かせる |
| 採点 | 自動採点（入力 → 「解答する」→ 正誤判定） | 学習効果が高い |
| 勘定科目入力 | 問題ごとの候補（6個）からプルダウン選択 | ネット試験に近く、実装もシンプル |
| 金額入力 | テキスト入力（数値） | — |
| 入力行数 | 借方・貸方とも固定3行、空欄行は無視 | 正解の行数を漏らさない／実装が単純 |
| 出題フロー | 1問ずつ連続出題（解答 → 正誤 → 次へ） | 一問一答のテンポ |
| 出題順 | 問1→問15の順番通り | シャッフルは将来拡張 |
| 永続化 | `localStorage`（正誤履歴・再開位置） | DB不要で復習の土台になる |

## 2級追加時に確定した設計（インタビューで確定）

| 項目 | 決定 | 理由 |
|---|---|---|
| 画面構成 | 級選択 → （2級のみ）問題集選択 → 出題 の3階層 | 2級は190問8セット。論点ごとに区切って学習・再開したい。3級は1セットなので選択画面をスキップ |
| 進捗表示 | 選択画面のカードに「解答済 n/N・正答率 xx%」 | 次にやるセットを選べる |
| 勘定科目の候補 | 正解の科目＋同じ問題集の他科目（ダミー）で6個。問題IDから作った固定シードで並べる | 元データに候補リストが無い。同じ論点の科目が混ざるのでダミーも紛らわしく、かつ毎回同じ並びになる |
| 進捗キー | `boki-progress-v3`。問題キーは `g2-03-q5` 形式、再開位置はセットごと | 問題IDは問題集内でしか一意でない（どのセットにも `q1` がある） |
| 旧データ | v3 が未作成のときだけ v2 を読み、`q1` → `g3-01-q1` へ変換 | 3級の解答履歴を失わせない |
| 本支店の9問 | 「（本店）」「（支店）」等に分割して別問題にする（20問 → 31問） | 「本店と支店の仕訳をそれぞれ答えなさい」形式。データ型と採点を変えずに済む |
| 入力行数 | `max(正解行数, 3)` で問題ごとに可変 | のれん・連結など4行必要な問題が3問ある。全問4行にすると1行仕訳でも欄が余って冗長 |
| 「仕訳なし」 | 専用チェックで解答させ、チェック時のみ正解 | 採点は「埋めた行を比較」なので、空欄提出が正解になってしまうと判断を問えない |

## 採点ロジック（[src/lib/grading.ts](../src/lib/grading.ts)）

- 借方・貸方それぞれを **「(科目, 金額) の集合」として順不同で比較**。
- 埋めた行（科目と金額の両方が有効）だけを対象にし、両側とも過不足なく一致すれば「正解」。
- 金額入力は正規化してから比較する（`parseAmount`）:
  - 全角数字 → 半角
  - カンマ・空白・`￥`・`円` を除去
  - 数字以外が残ればその行は無効扱い（`1,300` も `1300` も同一とみなす）

## コンポーネントの分け方

出題画面は「状態」と「見た目」を分けている。

- [useQuizSession](../src/lib/useQuizSession.ts) — 出題位置・入力・採点結果と、それを動かす操作（`submit` / `goTo` / `retry` など）を1か所に集める。永続化は `useProgress` に委ねる。
- [JournalQuiz](../src/components/JournalQuiz.tsx) — フックから受け取った値をパーツに配るだけ。ロジックを持たない。
- [components/quiz/](../src/components/quiz/) — 画面のパーツ。いずれも props だけで描画でき、状態を持たない。

パーツを足すときは `components/quiz/` に置き、状態が必要なら `useQuizSession` 側に寄せる。

## 2級データの生成

`docs/2級/*.md` は「問題文 + 借方/貸方の表」という決まった形なので、
[scripts/convert-2kyu.mjs](../scripts/convert-2kyu.mjs) で TypeScript へ変換している。

```bash
node scripts/convert-2kyu.mjs
```

- 出力先の `src/data/questions/g2-0N.ts` は**生成物**。直したいときは md かスクリプトを直す。
- 勘定科目の候補は md に無いので、正解の科目＋同じ問題集の他科目から6個を組み立てる。
  並びは問題IDのハッシュをシードにした疑似乱数で決めるため、再実行しても差分が出ない。
- `**解答（本店）**` のように解答表が複数ある問題は、その数だけ別問題に分割する。
- `**解答**：仕訳なし` は `noJournal: true` として書き出す。

## localStorage の保存内容

キー: `boki-progress-v3`

```jsonc
{
  // "セットID-問題ID" -> 解答履歴（古い順）
  "attempts": {
    "g3-01-q1": [{ "correct": false, "at": 1750000000000 }],
    "g2-03-q5": [{ "correct": true,  "at": 1750000100000 }]
  },
  // セットID -> 再開用の現在インデックス
  "currentIndex": { "g3-01": 7, "g2-03": 2 }
}
```

- 正答率は**各問の直近の試行**を基準に数える（何度も解き直せるため）。
- 旧キー `boki-progress-v2`（3級のみ・`q1` 形式・再開位置1つ）は、
  v3 がまだ無いときに限り読み込んで `g3-01-*` へ変換する。
- 「リセット」ボタンは**表示中の問題集ぶんだけ**消す。
- SSR とのハイドレーション不整合を避けるため、`localStorage` の読み込みは**マウント後**に行う。
